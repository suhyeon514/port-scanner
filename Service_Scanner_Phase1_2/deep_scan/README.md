# 프로젝트 개요

이 프로젝트는 네트워크 포트 스캔 결과를 기반으로 심층 분석을 수행하는 Python 스크립트로 구성되어 있습니다. 주요 구성 요소는 `bridge.py`와 `core` 폴더 내의 모듈들로 이루어져 있습니다. 이 문서에서는 각 파일의 역할과 코드 동작 흐름, 그리고 결과로 생성되는 JSON 파일에 대해 설명합니다.

---

## 파일 및 폴더 구조

```
.
├── bridge.py
└── core/
    ├── logic.py
    ├── scapy_engine.py
    └── __pycache__/
```

### 주요 파일 설명

- **bridge.py**: 프로젝트의 진입점으로, Nmap 스캔 결과를 읽고 `core` 모듈을 사용하여 심층 분석을 수행한 후 결과를 JSON 파일로 저장합니다.
- **core/scapy_engine.py**: Scapy를 사용하여 네트워크 패킷을 생성하고 전송하며, 원시 응답 데이터를 수집합니다.
- **core/logic.py**: 추론 로직을 담당하며, 포트 상태와 원시 응답 데이터를 기반으로 정책을 도출합니다.

---

## 코드 동작 흐름

### 1. Nmap 결과 파일 읽기

`bridge.py`는 `runs` 폴더에서 가장 최신의 `_final_report.json` 파일을 검색합니다. 이 파일은 Nmap 스캔 결과를 포함하고 있어, 분석의 입력 데이터로 사용됩니다.

### 2. Scapy를 통한 심층 테스트

`core/scapy_engine.py` 파일은 Scapy를 사용하여 네트워크 패킷을 생성하고 전송하며, 원시 응답 데이터를 수집하는 역할을 합니다. 이 파일의 주요 클래스와 메서드는 다음과 같습니다:

#### ScapyInferenceEngine 클래스

이 클래스는 대상 IP와 포트에 대해 다양한 테스트를 수행하여 포트 상태를 분석합니다. 기본적으로 3가지 테스트를 제공합니다:

1. **ACK 테스트**
   - 연결 없는 ACK 패킷을 전송하여 대상이 RST 패킷을 반환하는지 확인합니다.
   - 반환값:
     - `no-response`: 응답 없음
     - `rst-returned`: RST 패킷 반환
     - `icmp-type-{type}`: ICMP 패킷 반환 (타입 포함)

2. **SYN-ACK 테스트**
   - 예고 없는 SYN-ACK 패킷을 전송하여 대상이 응답하는지 확인합니다.
   - 반환값:
     - `responded`: 응답 있음
     - `no-response`: 응답 없음

3. **예약 비트 테스트**
   - TCP 헤더의 예약된 비트를 변조하여 패킷을 전송하고, 대상이 이를 처리하는지 확인합니다.
   - 테스트는 0~7번 비트에 대해 각각 수행되며, 반환값은 다음과 같습니다:
     - `pass`: 응답 있음
     - `drop`: 응답 없음

#### 예약 비트 테스트의 동작

TCP 헤더에는 6개의 플래그 비트 외에도 6개의 예약된 비트가 존재합니다. 이 비트는 일반적으로 사용되지 않으며, 특정 네트워크 장비나 방화벽이 이를 처리하는 방식을 테스트할 수 있습니다. `test_reserved_bits_full` 메서드는 다음과 같은 방식으로 동작합니다:

1. TCP 헤더의 예약된 비트를 0부터 7까지 순차적으로 설정합니다.
2. 각 패킷을 대상 IP와 포트로 전송합니다.
3. 응답 여부를 기록하여 반환합니다.

예제 결과:

```json
{
    "0": "pass",
    "1": "drop",
    "2": "drop",
    "3": "pass",
    "4": "drop",
    "5": "drop",
    "6": "pass",
    "7": "drop"
}
```

이 결과는 대상이 특정 예약 비트에 대해 응답하거나 패킷을 차단하는 방식을 보여줍니다.

### 3. 추론 로직 수행

`core/logic.py` 파일은 네트워크 응답 데이터를 기반으로 보안 정책을 추론하는 역할을 합니다. 이 파일의 주요 클래스와 메서드는 다음과 같습니다:

#### DecisionLogic 클래스

이 클래스는 Scapy를 통해 수집된 원시 응답 데이터를 분석하여 보안 정책과 네트워크 동작을 추론합니다. `infer_policy` 메서드는 입력된 테스트 결과와 프로토콜 정보를 바탕으로 결론과 이유를 반환합니다.

#### 주요 동작 과정

1. **공통 무응답 분석**
   - 모든 테스트 결과가 응답 없음(`no-response`) 또는 패킷 폐기(`drop`)로 나타날 경우, 강력한 보안 정책이 적용된 것으로 간주합니다.
   - 결론: "강력한 정적/상태 기반 차단 정책"
   - 이유: "모든 비정상 패킷에 대해 응답 없이 패킷을 폐기함."

2. **TCP 프로토콜 분석**
   - **심층 패킷 검사(DPI) 탐지**: 예약 비트가 포함된 패킷을 선별적으로 차단하는 경우, DPI가 작동 중인 것으로 추정합니다.
     - 결론: "심층 패킷 검증(DPI) 작동 중"
     - 이유: "표준 패킷(0)은 허용하나, 비표준 예약 비트를 탐지하여 차단함."
   - **표준 포트 닫힘**: ACK 테스트에서 RST 응답이 반환되면, 방화벽 간섭이 없는 상태로 간주합니다.
     - 결론: "표준 포트 닫힘 (방화벽 간섭 없음)"
     - 이유: "운영체제가 표준 RST 응답을 보냄."

3. **UDP 프로토콜 분석**
   - UDP 포트에서 TCP 응답이 반환되는 경우, 프로토콜 비의존적 정책이 적용된 것으로 간주합니다.
     - 결론: "프로토콜 비의존적 통과 정책"
     - 이유: "UDP 포트임에도 TCP 변조 패킷이 호스트 스택까지 도달함."

4. **ICMP 반환 분석**
   - ACK 테스트에서 ICMP 응답이 반환되면, 명시적 거부 정책이 적용된 것으로 간주합니다.
     - 결론: "명시적 거부 정책 (ICMP 반환)"
     - 이유: "보안 장비가 정책에 따라 패킷을 차단하고, 도달 불능 메시지를 반환함."

5. **기타 복합적 보안 정책**
   - 일반적이지 않은 응답 패턴이 나타날 경우, 복합적인 보안 정책이 적용된 것으로 간주합니다.
     - 결론: "복합적 보안 정책 탐지"
     - 이유: "특수한 보안 설정이나 네트워크 환경의 영향일 수 있음."

#### 예제 결과

입력된 테스트 결과에 따라 다음과 같은 추론 결과를 반환할 수 있습니다:

```json
{
    "conclusion": "심층 패킷 검증(DPI) 작동 중",
    "reason": "표준 패킷(0)은 허용하나, 비표준 예약 비트를 탐지하여 차단함."
}
```

### 4. 결과 저장

분석 결과는 다음과 같은 구조의 JSON 파일로 저장됩니다:

```json
{
    "target_ip": "192.168.1.1",
    "inference_details": [
        {
            "port": 80,
            "proto": "tcp",
            "nmap_state": "filtered",
            "raw_responses": ["response1", "response2"],
            "inferred_policy": "Allow",
            "reasoning": "The port responded with SYN-ACK packets."
        },
        {
            "port": 443,
            "proto": "tcp",
            "nmap_state": "closed",
            "raw_responses": [],
            "inferred_policy": "Deny",
            "reasoning": "No response received."
        }
    ]
}
```

## 결과 JSON 파일

### 파일명

`bridge.py`는 결과를 `runs/open_ports.json` 파일로 저장합니다.

### 주요 필드 설명

- **target_ip**: 분석 대상 IP 주소.
- **inference_details**: 각 포트에 대한 분석 결과 배열.
  - **port**: 포트 번호.
  - **proto**: 프로토콜(`tcp`, `udp` 등).
  - **nmap_state**: Nmap에서 감지한 포트 상태.
  - **raw_responses**: Scapy를 통해 수집된 원시 응답 데이터.
  - **inferred_policy**: 추론된 정책(`Allow`, `Deny` 등).
  - **reasoning**: 정책 도출의 근거.
